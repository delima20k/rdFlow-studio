📱 Exemplos de Uso

Câmera Remota PWA - Casos de uso reais e configurações

🎥 Caso 1: Streaming de Evento

Cenário

Você está fazendo live de um evento e quer uma câmera wireless para mostrar diferentes ângulos.

Configuração

Backend: Rodando em servidor VPS (ex: DigitalOcean)

PWA: Hospedado no Vercel

https://camera-remota.vercel.app/?api=https://meu-servidor.com&session=evento-live-001

Fluxo

  • Abra o estúdio no notebook: http://meu-servidor.com/studio.html
  • Clique em "📱 Câmera do Celular"
  • Mostre o QR Code para o celular
  • Celular conecta automaticamente
  • Use o celular para capturar ângulos diferentes

🎓 Caso 2: Aula Online

Cenário

Professor dando aula online quer mostrar experimentos práticos usando o celular como segunda câmera.

Configuração

Backend: Localhost (computador do professor)

PWA: GitHub Pages

Rede: Professor e celular na mesma WiFi

https://professor.github.io/camera-remota-pwa/?api=http://192.168.0.100:3000&session=aula-fisica-001
💡 Dica: Use o IP local do computador (ipconfig no Windows ou ifconfig no Linux/Mac) para conectar o celular.

🎙️ Caso 3: Podcast com Múltiplos Ângulos

Cenário

Gravação de podcast com 2 apresentadores. Um celular filma cada apresentador.

Configuração

Celular 1: Filma apresentador A

Celular 2: Filma apresentador B

Notebook: Estúdio recebe ambas as câmeras

Celular 1: https://camera.vercel.app/?api=https://servidor.com&session=podcast-cam-A
Celular 2: https://camera.vercel.app/?api=https://servidor.com&session=podcast-cam-B

No Estúdio

  • Adicione "📱 Câmera do Celular" duas vezes
  • Use IDs de sessão diferentes
  • Cada celular conecta em sua própria sessão
  • Grave ou transmita com múltiplos ângulos

📺 Caso 4: Entrevista Remota

Cenário

Entrevistador em um local, entrevistado em outro. Entrevistado usa o celular como câmera profissional.

Configuração

Backend: Servidor público (Heroku, Railway, Render)

PWA: Netlify

Conexão: Internet 4G/5G

https://camera-remota.netlify.app/?api=https://backend-entrevista.railway.app&session=entrevista-convidado-123
⚠️ Atenção: Para funcionar em redes diferentes (4G vs WiFi), configure um servidor TURN no backend ou use Firebase/Twilio para signaling.

🎮 Caso 5: Streaming de Gameplay com Câmera Facecam

Cenário

Streamer quer usar celular como facecam em vez de webcam USB.

Configuração

PC: Rodando OBS + Backend local

Celular: PWA conectado ao PC via WiFi

OBS: Captura janela do navegador com preview da câmera

https://camera-remota.vercel.app/?api=http://192.168.1.10:3000&session=facecam-stream

Integração com OBS

  • Abra http://localhost:3000/studio.html
  • Conecte celular via QR Code
  • No OBS: Adicionar → Captura de Janela do Navegador
  • Selecione a janela do estúdio
  • Crop apenas a área do preview da câmera

🛠️ Configurações Avançadas

Forçar Resolução HD

Edite index.html linha ~275:

const constraints = { video: { deviceId: deviceId ? { exact: deviceId } : undefined, facingMode: deviceId ? undefined : currentFacingMode, width: { ideal: 1280 }, // Altere aqui height: { ideal: 720 } // Altere aqui }, audio: true };

Desabilitar Áudio

Útil se você já tem microfone externo:

const constraints = { video: { /* configurações */ }, audio: false // Desabilita áudio };

Adicionar Marca D'água

Edite CSS em index.html:

.camera-container::after { content: "AO VIVO"; position: absolute; top: 20px; right: 20px; background: red; color: white; padding: 8px 16px; border-radius: 4px; font-weight: bold; z-index: 10; }

📊 Qualidade vs Banda

Resolução Qualidade Banda (upload)
640x480 (SD) Baixa ~1 Mbps
1280x720 (HD) Boa ~2.5 Mbps
1920x1080 (Full HD) Alta ~5 Mbps
💡 Recomendação: Use 720p para WiFi doméstico e 480p para 4G.
